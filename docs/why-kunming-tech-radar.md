# 为什么要做昆明技术机会雷达？

如果你在 GitHub 上搜索昆明相关的技术内容，会发现结果很少。

这并不必然说明昆明没有技术机会。更可能的情况是：本地的技术机会没有被结构化地表达出来。

一线城市的信息已经高度结构化。公司名单、岗位、活动、投资机构、开源项目、技术社区、创业空间，大量信息会自然流到 GitHub、公众号、招聘平台、媒体数据库和各种导航站里。一个开发者想了解北京、上海、深圳、杭州的技术生态，通常不缺入口。

但昆明和云南不是这样。

这里的机会更多散落在：

- 公司官网和很少更新的招聘页；
- 政府采购公告、公共资源交易平台、科技型中小企业名单；
- 公众号文章、园区动态、协会活动；
- PDF、公示名单、新闻稿；
- 老社区清单和熟人网络；
- 学校、老师、同学、客户之间的口口相传。

这些信息并不是没有价值，只是很难被普通学生、开发者、自由职业者和创业者看见。

## 把散落信息变成开放数据库

昆明技术机会雷达想做的事情很简单：

把这些散落的公开信息，整理成一个可以搜索、可以复核、可以被持续维护的开放数据库。

它不是公司排名，不是招聘中介，也不是商业背书。它更像一张本地技术生态的底图：先把节点标出来，再慢慢补来源、补方向、补区域、补机会类型。

当一家公司被收录时，我们关心的不是“它是不是最厉害”，而是：

- 它是否和技术、软件、数据、系统集成或数字化交付有关？
- 是否有官网、官方页、招聘页、政府公告或其他公开来源？
- 它可能属于哪个方向：AI、软件开发、系统集成、农业数字化、医疗信息化、文旅科技？
- 它对学生、开发者、自由职业者、创业者分别有什么参考价值？
- 这条信息的可信度是多少，还缺什么复核？

## 对学生：一个本地行业认知入口

很多学生找实习时，只能打开招聘 App 搜“昆明 Java”“昆明前端”“昆明实习”。这样看到的是岗位结果，不是行业地图。

如果先有一张本地技术机会地图，学生可以更早知道：

- 昆明有哪些软件公司、系统集成商、医疗信息化公司、农业数字化公司；
- 哪些公司至少有官网或官方页；
- 哪些方向可能适合毕业设计、课程项目、实习研究；
- 本地行业的真实结构是什么，而不是只看几个热门岗位。

## 对开发者：一张本地工作和合作地图

对已经工作的开发者来说，本地机会往往不是“有没有岗位”这么简单。

你可能想找：

- 本地技术团队；
- 外包或合作方；
- 行业数字化项目；
- 政企信息化和系统集成机会；
- 有技术需求但表达不充分的传统行业客户。

这些信息在招聘平台上并不完整。开源数据库能提供一个更底层的索引。

## 对自由职业者和创业者：客户线索和产业方向雷达

如果你想在昆明做技术服务，最重要的不是再做一个通用软件产品，而是找到真实场景：

- 哪些行业正在数字化？
- 哪些公司可能需要官网、系统、数据、AI、自动化、部署、运维？
- 哪些政府或产业方向反复出现？
- 哪些公司只是名字像科技公司，实际业务还需要复核？

这些问题不能靠想象解决，需要长期收集公开线索。

## 对城市：数字经济生态的微型索引

一个城市的技术生态，不只存在于大型园区、官方报告和招商材料里，也存在于很多小公司、小团队、小项目、校企合作、政府公告和开发者个人行动里。

如果这些信息长期不可见，外部会以为这里没有机会；本地年轻人也会更早把目光移向外地。

昆明技术机会雷达不是要证明昆明已经是技术中心，而是先做一件更基础的事：

让本地技术机会能被看见、被检索、被复核、被连接。

## 这个项目应该如何传播

这个项目最适合被这样使用：

- 学生把它当作找实习和了解本地行业的入口。
- 开发者把它当作本地工作、合作和社群地图。
- 自由职业者把它当作客户线索和交付方向雷达。
- 企业把它当作补充公开入口的地方。
- 维护者把它当作长期更新的城市技术数据库。

如果你知道一家昆明或云南的技术相关公司，不需要写代码，也可以提交一个 Issue。只要有公开来源，这张地图就能多一个节点。
