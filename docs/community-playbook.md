# 怎么一起维护这份清单

这份清单现在还不完整，也不应该装作很完整。

它更像一个公开草稿：先把昆明能找到的 IT、软件、信息化、互联网相关公司放到一个地方，再慢慢把来源补准。有人发现少了公司、官网变了、业务描述不准确，直接提 issue 或 PR 就行。

## 可以怎么介绍

发到群里或者社区时，不用说得太正式。可以直接这样写：

```text
我整理了一份昆明 IT 公司清单，放在 GitHub 上。

现在还不完整，有些公司也只是从公开名单里先列出来，后面需要继续补官网、招聘页和业务方向。如果你知道昆明还有哪些软件、互联网、信息化、AI、系统集成公司，可以直接开 issue 或 PR 补一下。

仓库地址：
https://github.com/tianmind-studio/kunming-it-companies
```

如果想再短一点：

```text
做了一个昆明 IT 公司开源清单，方便本地开发者和求职的人查。现在还有不少待核验条目，欢迎补公司、补官网、纠错。
https://github.com/tianmind-studio/kunming-it-companies
```

## 什么内容最值得补

- 公司官网或官方公众号
- 招聘主页，比如 Boss、拉勾、脉脉、公司官网招聘页
- 公司主营业务，尽量用公开来源里的说法
- 所在区县
- 公司已经停业、改名、合并或官网打不开的情况

## 不想收什么

- 未经核实的负面爆料
- 私人手机号、私人微信
- “昆明最强”“最好”“坑不坑”这类评价
- 没有来源的薪资、加班、裁员信息

这些信息不是不重要，只是不适合放在这个仓库里。这个项目先做一件简单的事：让“昆明有哪些技术相关公司”这件事更容易查、更容易补、更容易纠错。

## 贡献方式

- 少了一家公司：开 `Add company` issue
- 信息写错了：开 `Update company` issue
- 找到官网或官方来源：开 `Verify company` issue
- 熟悉 GitHub：直接改 `data/companies.json` 提 PR
