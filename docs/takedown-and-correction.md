# Takedown and Correction Policy / 更正与删除规则

这个项目只整理公开来源。若某条记录错误、过期、含有不适合公开的信息，欢迎请求更正或删除。

## 可以请求更正的情况

- 公司名称、区县、官网、来源链接错误。
- 链接失效或指向无关页面。
- 公司已经更名、注销、迁出或业务方向明显变化。
- 记录把弱来源误标成已核验。
- 活动、项目、社群状态过期。

## 可以请求删除的情况

- 记录包含私人联系方式、聊天截图或不应公开的信息。
- 记录无法找到任何公开来源。
- 记录与技术机会无关。
- 记录涉及无法公开核验的负面传闻、薪资、裁员、纠纷或内部消息。

## 如何提交

普通用户可以通过 [`../submit.html`](../submit.html) 复制模板并联系维护者；开发者可以通过 GitHub Issue 提交：

1. 说明要更正或删除的记录 ID / 名称。
2. 贴出公开来源链接。
3. 简短说明为什么需要更正。
4. 如果涉及隐私，不要在 Issue 中粘贴隐私内容本身，只说明问题类型。

维护者处理时会尽量保留一条中性的公开记录：例如把 `verified` 降级为 `community_pending`，或把具体项目改成“来源入口”。
