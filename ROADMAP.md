# Roadmap

Kunming Tech Radar 会从公司索引逐步扩展为本地技术机会资料库。

## Phase 1: 数据结构

- [x] 建立公司、岗位、活动、社群、政府项目五类数据表
- [x] 保留早期公司索引，作为迁移和复核基础
- [x] 为数据增加基础校验脚本
- [x] 明确 `data/companies.json` 是公司主数据源，`COMPANIES.md` 与 `data/companies.csv` 由脚本生成
- [x] 增加每个方向至少 5 条公开来源种子

## Phase 2: 可信来源

- [ ] 把待复核公司补充为官网或官方主页来源
- [x] 建立招聘平台入口的保守记录规则：只收公开主页/搜索入口，不编造岗位状态
- [x] 建立社群、活动、政府项目的来源入口表
- [x] 建立岗位和活动的过期/来源入口标记规则
- [x] 建立政府项目线索的来源格式：门户入口和具体公告分开记录
- [ ] 每条记录保留来源链接和核验日期
- [ ] 为每个方向补充至少 1 条非招聘平台来源入口，例如高校、园区、公共资源、政府或行业机构入口

## Phase 3: 机会雷达

- [x] 建立按区县、行业、机会类型浏览的静态页面
- [x] 从 JSON/CSV 数据驱动静态页面
- [ ] 支持按更新时间查看新增线索
- [ ] 增加中性的更新日志和数据质量报告
- [ ] 补充 GitHub Pages 分享预览图和元信息

## Phase 4: 公共协作与信任

- [x] 建立 Issue / PR 模板，让不会 GitHub 的用户也能提交公开来源
- [x] 建立公共复用、引用、更正和社区行为边界文档
- [ ] 建立固定复核节奏：每周处理 Issue、链接失效和高优先级待核验记录
- [ ] 公开记录重大数据修正、删除和来源升级原因
- [ ] 根据真实贡献反馈改进模板和校验脚本
